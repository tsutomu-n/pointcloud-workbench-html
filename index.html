01 // SYSTEM CAPABILITIES

Chunked Processing

Large-file pressure smoothing

Chunked LAS reads and LAZ chunk transfer into WASM heap reduce load spikes and unnecessary memory duplication.

Pre-Flight Evaluation

Pre-load risk and quality review

Header-first inspection exposes load path, risk level, source point count, and estimated render ratio before you commit to import.

Multi-Dimensional Viz

3D, 2D, slice, stats

Switch between 3D and 2D, inspect slices, review classification output, and validate results with the statistics panel in one browser tool.

02 // CONSTRAINTS & REQUIREMENTS

Requirements

  • Recommended: latest Chrome / Edge / Firefox
  • Required APIs: WebGL / File API / ArrayBuffer
  • Accepted formats: .las / .laz
  • Network: runtime loads three.js and laz-perf from CDNs

System Limits

  • LAS ceiling: above 3GB is rejected. 2GB to 3GB should start from LOW.
  • LAZ ceiling: decoder overhead narrows the safe range. above 2GB is rejected.
  • Auto classification: assistive only. It is not a survey-grade classification guarantee.
  • URL loading and some compatibility paths may still use full ArrayBuffer reads.
