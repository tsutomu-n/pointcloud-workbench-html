# トラブルシューティング

## アプリが起動しない

確認:

- ChromeまたはEdge最新版か
- ネットワーク接続があるか
- JS/WASMの取得がブロックされていないか
- 社内プロキシやセキュリティソフトでCDNが止められていないか

## LAS/LAZが読み込めない

確認:

- 拡張子が `.las` または `.laz` か
- ファイルサイズ上限を超えていないか
- LASは3GB超で拒否される
- LAZは2GB超で拒否される
- まずLOW品質で試す

Issueで報告する場合:

- ファイル選択画面の「診断情報をコピー」を押す
- コピーされたJSONをIssueへ貼り付ける
- 診断情報は手動コピーのみで、自動送信されない
- 診断情報にはCRS、取得品質、地表候補などの要約が含まれる
- LAS/LAZファイル本体、ローカルファイル名、元LAS座標配列は診断情報に含めない

現場状況や操作状態も共有する場合:

- 統計パネルの「作業メモコピー」を押す
- 表示状態、断面、計測、異常候補、取得品質、地表候補の要約JSONを貼り付ける
- 作業メモも手動コピーのみで、自動送信されない

## 画面が重い

対応:

1. LOW品質で読み込む。
2. 分類表示ではなく標高表示にする。
3. スライス表示をOFFにする。
4. 他のブラウザータブを閉じる。
5. Chrome/Edgeを再起動する。

## Geo Previewが使えない

確認:

- 点群にCRS情報があるか
- ネットワーク接続があるか
- 地図タイル取得がブロックされていないか
- Hosted High Performance ModeでCOEPにより外部タイルがブロックされていないか

## スクリーンショットに地図が入らない

外部地図タイルのCORS制約により、地図込みスクリーンショットが制限される場合があります。
点群のみのスクリーンショットを使ってください。

## 地表候補の confidence が低い

確認:

- ground分類点が少ない、または分類されていない可能性がある
- `p05推定` は分類が薄い場合の下位パーセンタイル推定であり、成果品DTMではない
- 勾配、土量、設計面比較の根拠に使わない
